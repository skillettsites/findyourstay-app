## 1. What this is
An **Airbnb-style, direct-booking directory + host-services platform** for independent stays. Travellers browse and book direct (no platform fees); hosts pay a recurring fee to be listed and, optionally, to have a **booking website built and hosted for them**. The operator never touches booking money (guests pay the host directly via the host's own Stripe/PayPal link).

## 3. Current pricing (host page `/host`)
Paid tiers **include a built booking website by default** (the host is assumed to need one). A **"I already have my own website"** tick removes the £120 website add-on and lowers the price. The **founding price (20% off, for life) is shown as the big headline**; the standard price is shown smaller and struck-through beneath, with the `FOUNDING20` code.

| Tier | Listing only (standard / founding) | + Website (standard / founding) |
|---|---|---|
| Free | £0 | n/a (list-only) |
| Standard | £79 / £63 | £199 / **£159** |
| Featured (most popular) | £149 / £119 | £269 / **£215** |
| Pro | £299 / £239 | £419 / **£335** |

Website add-on = **£120/yr** (we register a domain, build + host the site on Cloudflare, HTTPS, keep it indexed). Founding = 20% off the whole thing, forever.

**Pricing verdict (analysis done today):** NOT too expensive — it's the cheapest *done-for-you* booking-website option vs the market (Lodgify ~£380-760/yr, eviivo ~£400-700, Hostfully ~£1,200+, Freetobook £100-1,200, DIY Wix ~£180-300 + your time). Cash cost to serve a host ≈ **£12-16/yr** (domain + Stripe fee; hosting/email/AI marginal ≈ £0) → **~95% gross margin**; the real constraint is fulfilment time. The barrier to "no-brainer" is **trust in a new brand, not price** — so DON'T cut price. Recommended (not yet built): add a **money-back / "pay when you're happy" guarantee** and a **"what others charge" comparison row** on the host page, and keep the **commission-saved** framing ("less than the commission on ~3-4 bookings / one booking covers your year").

**Libs:** `src/lib/db.ts` (Supabase data layer, `createListing` with geocode+`fuzz()`), `src/lib/auth.ts` (@supabase/ssr server, `getUser`/`ensureHost`), `src/lib/supabase-browser.ts`, `src/lib/email.ts` (Resend), `src/lib/payments.ts` (Stripe billing + Connect), `src/lib/calendar.ts` (iCal engine), `src/lib/provision.ts` (Cloudflare domain provisioning — gated off), `src/lib/exampleStays.ts` (`EXAMPLE_TEMPLATES`).
**Key components:** `Header.tsx` (dropdown menu, auth-aware), `components/host/PricingCards.tsx` (the pricing logic in §3), `ExplainerFilm.tsx` + `ExplainerFeatures.tsx` (animated deep-dive: existing-site, build, no-commission, HTTPS, calendar sync, Google/Bing indexing, llms.txt/AI), `ConciergeChat.tsx`, `RoomGallery.tsx`, `ListingCard.tsx`, `site/StandaloneSite.tsx` (microsite renderer).

## 8. Business context (analysis done today, for continuity)
- **Verdict:** good, well-timed, well-built concept; clearly a better *business* than the old SEO/affiliate site (which has structurally weak economics). Right trend (anti-OTA-commission "book direct"), capital-light (never touch money), high-margin recurring revenue, AI-era positioning (llms.txt, grounded concierge).
- **Sell the SERVICE, not the directory:** lead with "we build you a booking website + get you found on Google/Bing/AI," which delivers value even before FYS has guest traffic — this sidesteps the marketplace cold-start. The directory is the funnel/bonus.
- **Market (UK):** ~481k short-term-let properties + ~25-30k B&Bs; demand growing (~94m guest nights/yr). Realistic solo-operator obtainable: 500-1,500 paying hosts → ~£90k-£375k/yr recurring.
- **Old site = top-of-funnel:** its only real asset is SEO surface; best use is to funnel guests into this app, not compete with it.
